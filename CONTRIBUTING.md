# Contributing

## Workflow

1. Pick an issue from the backlog.
2. Move it to `In Progress`.
3. Create a branch for the task.
4. Open a pull request when the work is ready.
5. Ask the other teammate for review.
6. Merge only after review and basic verification.

## Branch naming

- `feature/<short-name>`
- `fix/<short-name>`
- `docs/<short-name>`

## Pull request checklist

- The task is linked to an issue.
- The change is small and focused.
- The description explains what changed.
- The other teammate can review it quickly.
